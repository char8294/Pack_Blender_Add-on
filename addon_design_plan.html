Script Toolkit

Blender Add-on รวมเครื่องมือ FBX โดยแยกงาน Batch ไปทำใน Blender worker ตัวใหม่ และเก็บงานที่แก้ไข object ที่เลือกไว้ใน Blender ปัจจุบัน

แนวทางนี้ปกป้องไฟล์งานที่ผู้ใช้กำลังเปิดอยู่: งาน Batch จะไม่ล้าง Scene ของ Blender หลัก ขณะที่ Delete Faces by Material และ Clear Custom Properties ยังรันได้ทันทีบน selection ปัจจุบัน

1. คำตอบสั้น ๆ: เปิด Blender ใหม่เพื่อรัน Batch ได้ไหม?

ทำได้ — Add-on ใช้ bpy.app.binary_path เรียก executable ของ Blender ตัวเดียวกับที่กำลังใช้อยู่ แล้วเปิดเป็น process อิสระด้วย --factory-startup (ไฟล์เริ่มต้นใหม่) และสั่ง worker script ให้รัน job ที่เลือกไว้

โหมดแนะนำ: Background Worker

แนะนำ

เปิด Blender ตัวที่สองแบบไม่มีหน้าต่างด้วย --background --factory-startup งาน Batch ทำเสร็จแล้วปิดเอง จึงไม่รบกวนไฟล์งานหรือหน้าจอของผู้ใช้

โหมดทางเลือก: Visible Worker

ใช้เมื่อ debug

เปิด Blender ตัวที่สองแบบเห็นหน้าต่างด้วย --factory-startup แต่ไม่มี --background เหมาะสำหรับดู error/ทดสอบ ไม่เหมาะเป็นค่าเริ่มต้น เพราะผู้ใช้ต้องปิดหน้าต่างเอง

ไม่ควรใช้คำสั่ง “New File” ภายใน Blender หลักสำหรับงาน Batch เพราะต้องเปลี่ยนไฟล์/ล้าง Scene ในหน้าต่างที่ผู้ใช้กำลังทำงานอยู่ และเสี่ยงต่อข้อมูลที่ยังไม่ได้บันทึก

6. Current Scene Tools

Delete Faces by Material

แก้ Mesh โดยตรง

เลือก Mesh ใน Scene แล้วกำหนด Material Prefix (default SKIN_body) จากนั้นลบ faces และ material slots ที่ตรงกัน มี confirmation dialog บอกจำนวน object/face ที่จะเปลี่ยน

Clear Custom Properties

รันทันที

เลือกว่าจะลบ Object properties และ/หรือ Mesh Data properties ของ object ที่เลือก โดยเก็บ _RNA_UI ไว้ และมีผลลัพธ์แจ้งจำนวน property ที่ลบ

ทั้งสองตัวเป็น UNDO operator เพื่อให้กด Ctrl+Z คืนการแก้ไขใน Blender หลักได้

7. สถาปัตยกรรม Parent → Worker

Script Toolkit UI (Blender หลัก) │ validate options + สร้าง job file (.json ใน temp) ▼ subprocess.Popen([bpy.app.binary_path, "--background", "--factory-startup", "--python", worker.py, "--", job.json]) │ ▼ Blender Worker (Scene ว่าง) ── import/process/export ทุก FBX │ เขียน progress + result log (.json/.txt) ▼ Script Toolkit UI อ่านสถานะ → แสดงสำเร็จ / ข้าม / error
  • Parent Add-on ทำหน้าที่เลือก tool, เก็บ options, ตรวจ path, สร้าง job และเปิด process เท่านั้น
  • Worker เปิดใหม่ทุก batch, อ่าน job JSON, เรียก logic ของ tool, ล้างเฉพาะ scene ของตัวเอง และปิดเมื่อจบ
  • ใช้ bpy.app.binary_path เป็นค่าเริ่มต้น จึงเปิด Blender เวอร์ชันเดียวกับ Add-on; จะมี option เลือก executable เองสำหรับกรณีพิเศษ
การสื่อสารสถานะ: เนื่องจาก worker เป็น process แยกกัน UI หลักต้องอ่านไฟล์ progress/result ที่ worker เขียนไว้เป็นระยะ แทนการเรียก bpy ข้าม process โดยตรง

9. ลำดับพัฒนา

  1. สร้าง Add-on ชื่อ Script Toolkit: panel, dropdown, property และคำอธิบายแต่ละ tool
  2. สร้างระบบ job JSON + Blender Background Worker และหน้าสถานะ
  3. ย้าย Re-export FBX ก่อน พร้อม checkbox Smooth/Normals/Scale และทดสอบว่า Blender หลักไม่ถูกแตะ
  4. ย้าย Batch Tool อีก 3 ตัวไป worker โดยค่อย ๆ เพิ่ม validation และรายงาน error ต่อไฟล์
  5. ย้าย Delete Faces/Clear Properties เป็น Current Scene operators พร้อม Undo/confirmation
  6. ทดสอบกับ path ว่าง, FBX เสีย, output ซ้ำ, armature/material หาย และหยุด worker กลางงาน

ผลลัพธ์: Script Toolkit ใช้งานเหมือนเครื่องมือเดียว แต่จัดการงานหนักอย่างปลอดภัยใน Blender ใหม่ และคงสองเครื่องมือแก้ selection ให้คล่องตัวในไฟล์ที่ผู้ใช้กำลังเปิดอยู่
